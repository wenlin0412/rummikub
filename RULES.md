## 遊戲準備

- 玩家人數：2–4 人（含 AI）
- 每位玩家起始**摸 14 張牌**
- 剩餘牌翻蓋成為**牌池（Pool）**
- 第一回合由玩家索引 0（房主）開始

---

## 回合流程

每回合限時 **120 秒**。輪到你時只能選擇以下其中一種行動：

### 選項 A：出牌（Meld）

1. 從手牌選取牌塊，組成合法牌組出到桌面，或加入現有桌面牌組。
2. **破冰後**可自由拆解並重組桌面上任何牌組。
3. 回合結束前，桌面所有牌組必須全部合法，否則觸發罰則。
4. 點擊「結束回合」確認出牌。

### 選項 B：從牌池摸牌

- 摸 1 張牌，**回合立即結束**。
- 本回合若已出過牌，則不能改為摸牌。

---

## 破冰行動（初次出牌）

每位玩家**首次出牌**須符合額外限制：

- 新出到桌面的牌，其數字總和必須 **≥ 30 點**
- **所有鬼牌的數字值皆為 0**，不計入破冰點數（一般百搭與雙人百搭均不計）
- **只能使用手中的牌**，不得取用或依附現有桌面牌組
- 破冰成功後，`hasBrokenIce` 旗標設為 `true`，之後的回合才可重組桌面

> ❌ 嘗試依附桌面牌組或未達 30 點時，系統會提示錯誤並拒絕結束回合。

---

## 桌面重組

破冰後，你可以在回合中：

- **拆分**現有桌面牌組，取出其中部分牌塊至手上暫存
- **重新配置**桌面所有牌塊（取出、合併、重排皆可）
- 取出的桌面牌（`borrowedTileIds`）**必須在同一回合內全部歸還**至桌面合法牌組，不可帶回手中

> 若回合結束時仍有借用中的牌未歸還，系統會拒絕結束回合。

## 合法牌組

所有牌組最少需要 **3 張牌**。

### 群組（Group）

**同數字、不同顏色**，3 或 4 張。

| 條件 | 說明 |
|------|------|
| 數量 | 有效張數 3 或 4（雙人百搭算 2 個位置）|
| 數字 | 所有非鬼牌數字必須**完全相同** |
| 顏色 | 所有非鬼牌顏色必須**各不重複** |
| 限制 | **變色百搭不可用於群組** |

**有效張數計算：**
```
有效張數 = 一般牌數 + 普通百搭數 × 1 + 雙人百搭數 × 2
```

#### 群組合法範例

| 牌組 | 說明 |
|------|------|
| 🔴7 + 🔵7 + ⬛7 | 3 色相同數字 ✅ |
| 🔴7 + 🔵7 + 🟡7 + ⬛7 | 4 色相同數字 ✅ |
| 🔴7 + 🔵7 + 🃏普通百搭 | 普通百搭補 1 個位置，共 3 個有效張數 ✅ |
| 🔴7 + 🔵雙人百搭 | 雙人百搭算 2 個位置，共 3 個有效張數 ✅ |

#### 群組非法範例

| 牌組 | 原因 |
|------|------|
| 🔴7 + 🔵7 | 有效張數僅 2，不足 3 ❌ |
| 🔴7 + 🔵7 + 🔴9 | 數字不同（7 ≠ 9）❌ |
| 🔴7 + 🔴7 + 🔵7 | 紅色重複 ❌ |
| 🔴7 + 🔵7 + ⬛7 + 🔴8 + ⬛变色百搭 | 含變色百搭，群組不允許 ❌ |
| 🔴7 + 🔵雙人百搭 + 🔵雙人百搭 | 有效張數為 5，超過上限 4 ❌ |

---

### 順組（Run）

**同顏色、連續數字**，最少 3 個數字位置，最多 13 個（即 1–13 全部）。

| 條件 | 說明 |
|------|------|
| 數量 | 總槽位（`totalSlots`）3–13 |
| 顏色 | 所有非鬼牌必須**同一顏色** |
| 數字 | 必須形成連續的整數序列，範圍在 1–13 |

**總槽位計算：**
```
totalSlots = 普通百搭數 × 1 + 雙人百搭數 × 2 + 一般牌數
```

#### 順組合法範例

| 牌組 | 說明 |
|------|------|
| 🔵5 + 🔵6 + 🔵7 | 最基本順組 ✅ |
| 🔵5 + 🔵6 + 🔵7 + 🔵8 + 🔵9 | 5 張同色連續 ✅ |
| 🔵5 + 🔵6 + 🃏普通百搭 + 🔵8 | 普通百搭代替 7 ✅ |
| 🔵5 + 🔵6 + 🔵雙人百搭 + 🔵9 | 雙人百搭佔 7+8 兩個位置 ✅ |
| 🔵1 + 🔵2 + 🔵3 + … + 🔵13 | 13 張全色順組（最大合法）✅ |

#### 順組非法範例

| 牌組 | 原因 |
|------|------|
| 🔵5 + 🔵6 | 槽位僅 2，不足 3 ❌ |
| 🔵5 + 🔵6 + 🔴7 | 顏色不同（藍 ≠ 紅）❌ |
| 🔵5 + 🔵7 + 🔵8 | 數字不連續（缺 6）❌ |
| 🔵12 + 🔵雙人百搭 | 雙人百搭需 12+13+14，但 14 超出範圍 ❌ |

## 特殊鬼牌詳細規則

### 🔴 普通百搭（Joker）× 2

- **功能：** 可代替任何顏色與數字的一般牌（佔 1 個槽位）
- **可用於：** 群組、順組均可
- **手牌結算：** 留在手中罰扣 **30 分**

**取回規則（唯一可被替換的鬼牌）：**

1. **破冰後**才可取回（`hasBrokenIce = true`）
2. 從手牌選一張符合該位置條件的牌替換下普通百搭
3. 替換須通過 `valSet()` 驗證（替換後牌組仍合法）
4. 取回的普通百搭**必須在同一回合**搭配至少 2 張手牌重新出牌，不可保留在手中

---

### 🔵 雙人百搭（Double）× 2

- **功能：** 在順組中佔據 **2 個連續同色數字**位置；在群組中算 **2 個不同顏色**的位置
- **手牌結算：** 留在手中罰扣 **30 分**
- **無法被替換**（只有普通百搭才能被取回）

**順組中的限制：**
- 雙人百搭佔位的兩個數字必須都在 1–13 範圍內
- 位置 12 之後不可放雙人百搭（因為需要 12、13、14，而 14 超出範圍）
- 不可用於填補不連續的缺口

**群組中的限制：**
- 有效張數 = 一般牌 + 普通百搭 × 1 + 雙人百搭 × 2，不得超過 4

**破冰計點：** 雙人百搭的 `number` 值為 0，**不計入**破冰 30 點門檻

#### 合法範例

| 牌組 | 說明 |
|------|------|
| ⬛5 + ⬛6 + 🔵雙人百搭 + ⬛9 | 雙人百搭代替 ⬛7 + ⬛8 ✅ |
| 🔴7 + 🔵7 + 🔵雙人百搭 | 群組，雙人百搭算 2 個位置（共 4 個有效張數）✅ |

#### 非法範例

| 牌組 | 原因 |
|------|------|
| ⬛11 + ⬛12 + 🔵雙人百搭 | 雙人百搭需 13+14，14 超出範圍 ❌ |
| 🔴7 + 🔵雙人百搭 + 🔵雙人百搭 | 有效張數 = 1 + 2 + 2 = 5，超過群組上限 4 ❌ |

---

### ⬛ 變色百搭（Color Change）× 2

- **功能：** 將順組分成**顏色不同的兩段**，自身佔據其中一個數字位置
- **只能用於順組**，群組完全不可用
- **手牌結算：** 留在手中罰扣 **30 分**
- **無法被替換**

**驗證邏輯：**
1. 一組牌中**只能有 1 張**變色百搭（`ccIdx.length !== 1` 即非法）
2. 變色百搭不可在牌組的**最左端或最右端**（左右兩側都必須有牌）
3. 左側所有非鬼牌須為**同一顏色**，右側所有非鬼牌須為**另一種顏色**
4. 左側顏色**不得等於**右側顏色
5. 左側 + 右側的數字（加上變色百搭本身佔的位置）必須能拼成**連續整數序列**
6. 左側所有非鬼牌數字必須**小於**變色百搭所在位置的數字
7. 右側所有非鬼牌數字必須**大於**變色百搭所在位置的數字

#### 合法範例

| 牌組 | 說明 |
|------|------|
| 🔴3 + 🔴4 + ⬛變色 + 🔵6 + 🔵7 | 變色佔 5，左側紅 3–4，右側藍 6–7 ✅ |
| 🔴1 + 🔴2 + 🔴3 + ⬛變色 + 🔵5 | 變色佔 4，左紅 1–3，右藍 5 ✅ |
| 🔴3 + 🃏普通百搭 + ⬛變色 + 🔵6 + 🔵7 | 普通百搭代替 🔴4，變色佔 5 ✅ |

#### 非法範例

| 牌組 | 原因 |
|------|------|
| 🔴3 + 🔴4 + ⬛變色 + 🔴6 + 🔴7 | 左右顏色相同（都是紅）❌ |
| ⬛變色 + 🔴3 + 🔴4 | 變色在最左端，左側無牌 ❌ |
| 🔴3 + 🔴4 + ⬛變色 | 變色在最右端，右側無牌 ❌ |
| 🔴3 + 🔴4 + ⬛變色1 + ⬛變色2 + 🔵6 | 兩張變色百搭 ❌ |

---

### 🟡 鏡射百搭（Mirror）× 2

- **功能：** 置於牌組**正中間**，使左右形成鏡射（回文）結構
- **限制：** 牌組長度必須為**奇數**（偶數長度必定非法）
- **手牌結算：** 留在手中罰扣 **0 分**（鏡射百搭不罰分）
- **無法被替換**
- **破冰計點：** 數字值 0，不計入破冰點數

**鏡射順組驗證邏輯：**
1. 鏡射百搭自動移至正中間位置（`normalizeMirrorSet`）
2. 所有非鬼牌須為**同一顏色**
3. 左右以鏡射百搭為軸，`left[i]` 與 `right[右數第 i 個]` 若都有數字，則數字必須**相等**
4. 左側（不含鏡射）的非鬼牌數字必須形成**連續遞增序列**
5. 所有數字必須在 1–13 範圍內

> 換言之，合法的鏡射順組形如：**3, 4, 5, [鏡射], 5, 4, 3**（左升右降的山形）

**鏡射群組驗證邏輯：**
1. 所有非鬼牌須為**相同數字**
2. `left[i]` 與 `right[右數第 i 個]` 若都有顏色，則顏色必須**相同**
3. 左側非鬼牌中**不得有重複顏色**

> 換言之，合法的鏡射群組形如：**紅7, 藍7, [鏡射], 藍7, 紅7**（左右顏色順序鏡射）

#### 合法範例

| 牌組 | 類型 | 說明 |
|------|------|------|
| 🔴3 + 🔴4 + 🟡鏡射 + 🔴4 + 🔴3 | 順組 | 左側 3,4 右側鏡射 4,3 ✅ |
| 🔴5 + 🔴6 + 🔴7 + 🟡鏡射 + 🔴7 + 🔴6 + 🔴5 | 順組 | 7 張鏡射順組 ✅ |
| 🃏百搭 + 🔴4 + 🟡鏡射 + 🔴4 + 🃏百搭 | 順組 | 普通百搭補位，中心對稱 ✅ |
| ⬛7 + 🔵7 + 🟡鏡射 + 🔵7 + ⬛7 | 群組 | 左黑藍對應右藍黑（鏡射）✅ |
| 🔴7 + 🔵7 + 🟡鏡射 + 🔵7 + 🔴7 | 群組 | 左紅藍對應右藍紅（鏡射）✅ |

#### 非法範例

| 牌組 | 原因 |
|------|------|
| 🔴3 + 🟡鏡射 + 🔴5 | 3 ≠ 5，左右不對稱 ❌ |
| 🔴3 + 🔴4 + 🟡鏡射 + 🔴3 + 🔴4 | 4 ≠ 3（右邊第 1 個 4 對應左邊第 2 個 4 才對）❌ |
| 🔴3 + 🔴4 + 🟡鏡射 + 🔴4 | 偶數長度（4 張），無正中心 ❌ |
| ⬛7 + 🔵7 + 🟡鏡射 + ⬛7 + 🔵7 | 左黑藍對應右黑藍，並非鏡射（應為右藍黑）❌ |
| 🔴7 + 🔴7 + 🟡鏡射 + 🔴7 + 🔴7 | 群組左側有重複顏色（兩張紅）❌ |

## 計時規則

每回合限時 **120 秒**（AI 回合不計時）。

| 時間到時的狀態 | 處置方式 |
|---|---|
| 本回合**已出過牌** | 撤銷所有動作，從牌池摸 1 張，回合結束 |
| 本回合**尚未出牌** | 強制從牌池摸 1 張，回合結束 |

## 罰則

### 無效桌面罰則（出牌後桌面含非法牌組）

觸發時機：點擊「結束回合」但桌面有不合法牌組。

處置方式：
1. **還原**桌面與所有手牌至本回合開始前的狀態
2. 從牌池**額外摸 3 張牌**
3. 回合結束，換下一位玩家

### 破冰限制違規

- 嘗試在破冰前使用桌面牌組 → 拒絕，提示「破冰限用手牌」
- 破冰點數不足 30 → 拒絕，提示當前點數

## 遊戲結束與計分

### 一般勝利（有人出完手牌）

- 出完手牌的玩家**獲勝**
- 勝者得分 = **所有落敗玩家手牌點數總和**
- 各落敗玩家得分 = **負自己手牌點數**

### 牌池耗盡（所有玩家連續都只摸牌未出牌）

當連續 N 輪（N = 玩家人數）全部玩家都僅摸牌未出牌，觸發牌池耗盡結束：

- **手牌點數最低**的玩家獲勝
- 勝者得分 = Σ（各落敗玩家手牌 − 最低手牌點數）
- 各落敗玩家得分 = −（自己手牌 − 最低手牌點數）

### 手牌計分方式

| 牌的種類 | 手中剩餘時的點數 |
|----------|----------------|
| 一般牌（1–13） | 牌面數字 |
| 🔴 普通百搭 | **−30 分** |
| 🔵 雙人百搭 | **−30 分** |
| ⬛ 變色百搭 | **−30 分** |
| 🟡 鏡射百搭 | **0 分**（不罰分）|
